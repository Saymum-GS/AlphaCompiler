# Alpha Compiler

## Overview
Alpha is a simple custom programming language with support for:
- Integer (`num`), float (`decimal`), and string (`text`) variable declarations.
- Input (`read`) and output (`show`) operations.
- Single-line (`alp>`) and multi-line (`alp{ }alp`) comments.

This project implements the full compiler using **Flex** (lexer) and **Bison** (parser) in **C**.

---

## Project Structure

```

alpha/
├── alpha.l                 # Lexer definitions
├── alpha.y                 # Parser and grammar rules
├── alpha.tab.c             # Generated by Bison
├── alpha.tab.h             # Generated by Bison
├── lex.yy.c                # Generated by Flex
├── alpha.exe               # Compiled executable
├── Makefile                # Build instructions
├── sample_alpha.alpha      # Sample program demonstrating features
├── sample_alpha_eval.alpha # Teacher-friendly evaluation program
└── README.md               # This file

````

---

## Requirements

- Windows or Linux system
- GCC compiler with `flex` and `bison` installed
- Make utility

---

## Build Instructions

1. Open terminal/PowerShell in the `alpha` project directory.

2. Clean any old build files:

```powershell
make clean
````

3. Generate parser and lexer:

```powershell
bison -d alpha.y      # produces alpha.tab.c and alpha.tab.h
flex alpha.l          # produces lex.yy.c
```

4. Compile the project:

```powershell
gcc -Wall -g -o alpha alpha.tab.c lex.yy.c -lfl
```

5. After compilation, `alpha.exe` (Windows) or `alpha` (Linux) will be created.

---

## Running Programs

1. Open terminal in the project directory.

2. Run your Alpha program:

```powershell
./alpha sample_alpha.alpha
```

or for evaluation programs:

```powershell
./alpha sample_alpha_eval.alpha
```

3. Follow any input prompts in the program. Output and execution results will be displayed in the terminal.

## Sample Program Evaluation

### Step 1: Variable Declaration

```alpha
num a;          # Integer variable
decimal b;      # Float variable
text c;         # String variable
```

**Explanation:** Declares three variables of different types. Compiler will show confirmation messages.

### Step 2: Initial Assignment

```alpha
a = 10;
b = 3.14;
c = "HelloAlpha";
```

**Explanation:** Assigns initial values to each variable.

### Step 3: Display Initial Values

```alpha
show("Initial Values:");
show(a);
show(b);
show(c);
```

**Explanation:** Prints the initial values. `show` can display both literals and variable values.

### Step 4: Arithmetic and Reassignment

```alpha
a = a + 5;
b = 6.28;
c = "Updated";
```

**Explanation:** Demonstrates arithmetic on integers and reassignment of decimal and string variables.

### Step 5: Display Updated Values

```alpha
show("Updated Values:");
show(a);
show(b);
show(c);
```

**Explanation:** Confirms that arithmetic and reassignment worked correctly.

### Step 6: Input from User

```alpha
show("Enter new values:");
read(a);
read(b);
read(c);
```

**Explanation:** Prompts user to input new values for all variable types.

### Step 7: Display Final Values

```alpha
show("Final Values:");
show(a);
show(b);
show(c);
```

**Explanation:** Shows the user-provided values to confirm correct input handling.

### Step 8: Comment Tests

```alpha
alp> This is a single-line comment

alp{
This is a
multi-line comment
}alp
```

**Explanation:** Tests both single-line and multi-line comment parsing.

### Step 9: Complex Arithmetic

```alpha
a = (a + 10) * 2;
b = b / 2.0;
show("Arithmetic Test:");
show(a);
show(b);
```

**Explanation:** Evaluates parentheses and combination arithmetic expressions.

---

## Notes

* Do **not** use `#` style comments; they are not supported.
* Always save Alpha programs with `.alpha` extension.
* Multi-line expressions should use parentheses for clarity.
